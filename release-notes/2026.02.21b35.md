Fix shared lock-trait field-name variants so Last Action, Auto-Lock, Auto-Lock Duration, and Tamper no longer show unavailable.

## Highlights
- Normalize lock trait decoding across camelCase and snake_case protobuf variants for:
  - `BoltLockTrait`
  - `BoltLockSettingsTrait`
  - `BoltLockCapabilitiesTrait`
  - `TamperTrait`
- Update cached auto-relock settings extraction to accept shared protobuf field-name variants.
- Preserve prior shared protobuf import and metadata compatibility fixes.
- Add regression coverage for mixed field naming in lock action, auto-relock, tamper, and structure decoding.

## Why
- After fixing shared protobuf import and metadata decoding, some device controls and diagnostics still appeared unavailable because several lock-related trait handlers only read one generated field-name style.
- On systems using a shared protobuf tree, that left valid trait data unread even though the stream stayed connected.

## Validation
- `./.venv/bin/python -m unittest tests/test_entity.py tests/test_protobuf_handler.py tests/test_api_client.py tests/test_protobuf_compat.py`
- `./.venv/bin/python -m compileall custom_components/nest_yale_lock tests/test_protobuf_handler.py tests/test_api_client.py`

## Breaking changes
- None intended. This beta only broadens field-name compatibility for shared lock-trait decoding.
