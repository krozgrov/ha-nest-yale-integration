Fix shared protobuf imports and keep Home Assistant area placement user-controlled during beta testing.

## Highlights
- Reuse an installed shared protobuf tree when available so this integration can coexist with another custom Nest integration that ships the same generated protobuf packages.
- Keep the issue #31 area behavior fix: stop creating or reassigning Home Assistant areas from Nest `where_label`.
- Preserve `where_label` as reference metadata in entity attributes.
- Add regression coverage for both shared-protobuf imports and non-mutating area behavior.

## Why
- Beta `2026.02.21b31` fixed the area ownership behavior, but some test systems still failed to load the integration when another custom Nest integration was installed because duplicate protobuf descriptors were being registered.
- This beta keeps the area fix intact and adds an import compatibility path so both integrations can load in the same Home Assistant instance.

## Validation
- `./.venv/bin/python -m unittest tests/test_entity.py tests/test_protobuf_handler.py tests/test_api_client.py tests/test_protobuf_compat.py`
- `./.venv/bin/python -m compileall custom_components/nest_yale_lock tests/test_protobuf_compat.py tests/test_protobuf_handler.py tests/test_api_client.py tests/test_entity.py`

## Breaking changes
- None intended. This beta preserves existing behavior changes from `2026.02.21b31` and adds a protobuf coexistence hotfix.
