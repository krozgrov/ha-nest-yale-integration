Add an experimental structure-side guest-create path to test plaintext Nest guest pincodes.

## Highlights
- Add `experimental_create_guest`, a separate service that sends `nest.trait.guest.GuestsTrait.CreateGuestRequest` on the structure resource using plaintext passcode bytes.
- Keep the existing `set_guest_passcode` flow unchanged and safe; this new path is isolated for targeted testing.
- Use the newly exposed `GuestsTrait` metadata to detect whether a new guest appears after the command is accepted.

## Validation
- `python3 -m compileall custom_components/nest_yale_lock`
- `python3 -m unittest tests/test_passcode_crypto.py tests/test_passcode_utils.py`

- None.
