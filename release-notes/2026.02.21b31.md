Stop recreating or reassigning Home Assistant areas from Nest `where_label` so manual area placement persists.

## Highlights
- Remove automatic Home Assistant area creation from Nest location metadata.
- Stop overwriting manual `area_id` assignments during later device-registry updates.
- Keep Nest `where_label` available in entity attributes for reference.
- Add focused unit coverage for the non-mutating area behavior.

## Why
- Issue #31 reported that deleting or manually reassigning a Home Assistant area did not stick because the integration recreated the area on the next refresh.
- Home Assistant should own final area placement; Nest location metadata is useful context, but it should not mutate HA area registry state.

## Validation
- `./.venv/bin/python -m unittest tests/test_entity.py`
- `./.venv/bin/python -m compileall custom_components/nest_yale_lock tests/test_entity.py`

## Breaking changes
- None intended. Existing Home Assistant area assignments are preserved; the integration simply stops forcing future area changes from Nest metadata.
