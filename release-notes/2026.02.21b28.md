Log successful `SendCommandResponse` operation details so field tests can inspect why provisional guest creates disappear before any durable Nest state change.

## Highlights
- Parse successful `nestlabs.gateway.v1.SendCommandResponse` payloads and log trait-operation summaries, including progress, status, publisher accepted state version, embedded command type URLs, and resource ids found in detail payloads.
- Include summarized response operations in `experimental_create_guest` acceptance and no-observable-change logs.
- Add unit coverage for the new success-path response parser.
- Record the `b27` result more precisely: the transient `Test1` guest exists only in `accepted` guest state and is retracted before any durable guest/user-access/pincode change appears.

## Why
- `b27` proved the create path is not reaching durable state, but the only gateway-side insight on success was a regex hint that `CreateGuestResponse` appeared somewhere in the raw payload.
- This build exposes the structured `SendCommandResponse.traitOperations` metadata so the next field run can show whether Nest is reporting a hidden status/detail on the successful command path.
