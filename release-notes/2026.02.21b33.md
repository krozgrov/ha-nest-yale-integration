Fix mixed protobuf field-name variants so shared DeviceIdentityTrait decoding works during observe streaming.

## Highlights
- Normalize `DeviceIdentityTrait` field access across snake_case and camelCase protobuf variants.
- Fix observe-stream crashes when shared protobuf modules expose fields like `serialNumber` and `fwVersion` instead of `serial_number` and `fw_version`.
- Keep the shared protobuf import hotfix and the Home Assistant area ownership fix from prior betas.
- Add regression coverage for camelCase `DeviceIdentityTrait` decoding.

## Why
- Beta `2026.02.21b32` fixed protobuf descriptor collisions, but some shared protobuf variants still exposed `DeviceIdentityTrait` fields with different generated attribute names.
- That caused observe processing to crash while decoding lock metadata, even though the integration imported successfully.

## Validation
- `./.venv/bin/python -m unittest tests/test_entity.py tests/test_protobuf_handler.py tests/test_api_client.py tests/test_protobuf_compat.py`
- `./.venv/bin/python -m compileall custom_components/nest_yale_lock tests/test_protobuf_handler.py`

## Breaking changes
- None intended. This beta only broadens protobuf field-name compatibility in metadata decoding.
