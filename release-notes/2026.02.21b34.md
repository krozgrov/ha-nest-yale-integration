Fix shared StructureInfoTrait field-name variants so observe streaming no longer crashes on `legacyId`.

## Highlights
- Normalize `StructureInfoTrait` legacy structure id access across snake_case and camelCase protobuf variants.
- Fix observe-stream crashes when shared protobuf modules expose `legacyId` instead of `legacy_id`.
- Keep prior shared protobuf import fixes and area ownership behavior intact.
- Add regression coverage for camelCase structure metadata decoding.

## Why
- Beta `2026.02.21b33` fixed shared `DeviceIdentityTrait` field-name differences, but some shared protobuf variants still exposed `StructureInfoTrait` metadata with camelCase fields.
- That caused observe processing to crash while applying structure metadata after the stream had already started successfully.

## Validation
- `./.venv/bin/python -m unittest tests/test_entity.py tests/test_protobuf_handler.py tests/test_api_client.py tests/test_protobuf_compat.py`
- `./.venv/bin/python -m compileall custom_components/nest_yale_lock tests/test_protobuf_handler.py`

## Breaking changes
- None intended. This beta only broadens protobuf field-name compatibility for shared structure metadata decoding.
