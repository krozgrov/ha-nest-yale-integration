Speed up startup companion backfill and defer companion entity creation until source fields exist, so lock-related entities appear quickly without temporary unavailable states.

## Highlights
- Retry startup companion-field backfill within seconds instead of waiting on slow fallback timing.
- Defer creating companion entities until their source fields are present:
  - Last Action
  - Auto-Lock
  - Auto-Lock Duration
  - Tamper
- Preserve the prior shared protobuf compatibility and area ownership fixes.
- Add regression coverage for the non-null source-field gating helper.

## Why
- Beta `2026.02.21b36` aligned companion entity availability with the push stream, but some systems still showed companion entities as unavailable for a while during startup.
- The underlying issue was a combination of partial initial observe data and creating companion entities before their source fields had arrived.

## Validation
- `./.venv/bin/python -m unittest tests/test_entity.py tests/test_protobuf_handler.py tests/test_api_client.py tests/test_protobuf_compat.py`
- `./.venv/bin/python -m compileall custom_components/nest_yale_lock tests/test_entity.py`

## Breaking changes
- None intended. Companion entities are now added only when their source fields exist, which avoids temporary unavailable states during startup.
